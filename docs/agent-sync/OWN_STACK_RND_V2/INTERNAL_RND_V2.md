## Critical Gap

The current actor work should be treated as a catalog foundation, not a workflow-complete SaaS. The missing layer is a native actor runtime:

1. `ActorSpec` registry: input schema, output dataset schema, required env names, base family, allowed providers, skip rules.
2. Shared base runners per family: one base runner per actor type/category.
3. Thin actor adapters: actor-specific URL/query mapping, selectors/API endpoint variants, output normalization.
4. Run endpoint: `POST /api/v1/actors/{actor_id}/runs` must execute the native stack, not redirect to Apify.
5. Result model: task/run/result/artifact records must use the existing SaaS database patterns.
6. Test matrix: one contract test per base family and sampled actor smoke tests per family, plus missing-key skip behavior.
